# Documentation Index and Governance

Last Updated: April 28, 2026  
Status: Active

This folder is organized to keep onboarding clear and avoid documentation sprawl.

## Folder Layout

- `START_HERE.md`
  - Primary onboarding and execution path for architects, BAs, developers, and coding agents.
- `setup.md`, `database-setup.md`, `architecture.md`, `api.md`, `security.md`, `security-checklist.md`, `requirement-traceability-matrix.md`
  - Active delivery documentation used for current implementation work.
- `reference/`
  - Background and strategic reference materials that support planning and review.
- `archive/`
  - Historical snapshots retained for context only. Not authoritative for current state.

## Governance Rules

- Keep exactly one source of truth per topic.
- Prefer updating an existing active document over creating a new one.
- Move stale status docs to `archive/` instead of deleting institutional context.
- Keep root-level documentation minimal: entrypoint, core spec, and delivery backlog.
- Update links when moving files.

## Document States

- `Authoritative`: required for implementation decisions and execution.
- `Supporting`: useful context but not the primary source of truth.
- `Archived`: historical only.
